Multi-Account + CI/CD Promotion

Extended the site into a mgmt/eng/test/prod org with role assumption via AWS Vault. CI/CD runs in GitHub Actions using OIDC (no long-lived keys). The pipeline builds once (SAM ‚Üí S3 artifact with checksums) and promotes the same artifact across environments with human approvals. Reliability is baked in: environment drift is prevented, parameters are consistent via SSM Parameter Store, and rollback is instant by re-deploying the last good artifact.
