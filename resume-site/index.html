This project began as the Cloud Resume Challenge, where I implemented a visitor counter backed by a RESTful API (API Gateway ‚Üí Lambda ‚Üí DynamoDB) and deployed a static site on CloudFront/S3 with HTTPS. Everything is defined as Infrastructure as Code using AWS CloudFormation / SAM.

I then extended it into a multi-account AWS setup (mgmt, eng, test, prod) with GitHub Actions CI/CD, environment protections (approvals for test/prod), and guardrails for cost & security. It mirrors how cloud-native teams ship: build once, promote safely, and roll back quickly when needed.

Multi-Account + CI/CD Promotion

Built a full mgmt/eng/test/prod org with role assumption via AWS Vault. CI/CD runs in GitHub Actions using OIDC (no long-lived keys). The pipeline builds once (SAM ‚Üí S3 artifact with checksums) and promotes the same artifact through environments with human approvals for test & prod. Rollback is fast by re-deploying the last known good artifact.

Security & Reliability

Enforced least-privilege IAM roles per account, IaC-only deployments (no console drift), and environment protections to gate high-risk changes. Config is parameterized via SSM so the same build moves across stages consistently.
