Multi-Account + CI/CD Promotion

Extended the build into a multi-account AWS setup (mgmt / eng / test / prod) under AWS Organizations, with role assumption via AWS Vault and MFA. CI/CD runs in GitHub Actions (OIDC, no long-lived keys). The pipeline builds once (SAM â†’ S3 artifact with checksums) and promotes the same artifact through environments with approvals. Config is centralized in SSM Parameter Store (AWS Systems Manager) to keep parameters consistent across stages. Supporting glue handled via lightweight shell scripts.

This mirrors common SaaS multi-account setups: ENG for rapid iteration, TEST as controlled pre-prod with human approvals and integration checks, and PROD gated by promotion of the exact, immutable artifact.
