Cloud Resume Challenge

How I built a full-stack, serverless, CI/CD-driven resume site on AWS

CloudResumeChallenge.dev Live Site GitHub

The Cloud Resume Challenge is a hands-on spec for proving cloud fundamentals end-to-end. I implemented the baseline and then pushed it further with infrastructure-as-code, automated testing, and a promotion pipeline.

Architecture at a glance

Front end

  • Static site (HTML/CSS/JS) in S3, distributed by CloudFront.
  • HTTPS via ACM; custom domain reubenmulholland.com on Route 53.
  • Visitor counter calls a backend API endpoint.

Back end

  • API Gateway REST endpoints for the counter.
  • Lambda (Python + boto3) reads/writes DynamoDB.
  • IAM least-privilege and explicit CORS to the site domain.
Everything as code. The entire stack is defined in AWS SAM (CloudFormation) YAML—no console drifting.

CI/CD pipeline

GitHub Actions runs build → test → deploy. Unit tests (PyTest) cover Lambda logic; integration tests call the API; E2E checks verify the counter on the live site. The pipeline packages SAM, uploads artifacts, then applies the stack. I also experimented with build-once promotion through ENG → TEST → PROD with approvals and rollback.

Security

  • Hardened IAM and S3 policies; explicit CORS.
  • No long-lived keys: GitHub → AWS via OIDC; MFA on all accounts.
  • Local auth via AWS Vault; nothing sensitive in source control.

What I learned

  • How DNS, certificates, CDNs, APIs, and IaC fit together in practice.
  • Tests + CI/CD create confidence (and make rollbacks trivial).
  • Why multi-account (multi-environment) promotion improves safety and parity.
